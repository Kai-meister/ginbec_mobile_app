b# GINBEC Mobile App — Project Guide

## Project Structure

```
my_company_app/
├── android/          ← auto-generated by Flutter
├── ios/              ← auto-generated by Flutter
├── web/              ← auto-generated by Flutter
├── pubspec.yaml      ← app config and dependencies
└── lib/              ← ALL your code lives here
    ├── main.dart                  ← app entry point
    │
    ├── config/                    ← theme, colors, constants
    │   ├── app_colors.dart        ← primary, secondary colors etc.
    │   ├── app_text_styles.dart   ← font sizes, weights
    │   ├── app_theme.dart         ← full app theme setup
    │   └── app_constants.dart     ← shared constant values
    │
    ├── assets/                    ← images, icons, fonts
    │   ├── images/
    │   ├── icons/
    │   └── fonts/
    │
    ├── models/                    ← data classes / structures
    │   ├── user_model.dart
    │   ├── employee_model.dart
    │   └── task_model.dart
    │
    ├── services/                  ← API calls, auth, database
    │   ├── api_service.dart
    │   ├── auth_service.dart
    │   └── database_service.dart
    │
    ├── widgets/                   ← reusable UI components
    │   ├── custom_button.dart
    │   ├── custom_card.dart
    │   └── loading_spinner.dart
    │
    └── screens/                   ← one folder per page
        ├── login/
        │   ├── login_screen.dart
        │   └── widgets//
        ├── home/
        │   ├── home_screen.dart
        │   └── widgets/
        ├── dashboard/
        │   ├── dashboard_screen.dart
        │   └── widgets/
        ├── employees/
        │   ├── employees_screen.dart
        │   └── widgets/
        ├── tasks/
        │   ├── tasks_screen.dart
        │   └── widgets/
        └── profile/
            ├── profile_screen.dart
            └── widgets/
```

## Conventions

- **Screens** live in `lib/screens/<feature>/`, one folder per page
- **Reusable widgets** go in `lib/widgets/`, screen-specific widgets go in `lib/screens/<feature>/widgets/`
- **Models** are plain data classes in `lib/models/`
- **Services** handle all external communication (API, auth, database) in `lib/services/`
- **Config** holds only theme, colors, text styles, and constants — no logic
- **Assets** (images, icons, fonts) are declared in `pubspec.yaml` and stored under `lib/assets/`
